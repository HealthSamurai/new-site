Integration of MedClient with AS400, Meditech and GE

Within the scope of the project of development and deployment of MedClient EHR in Verdugo Hills Hospital, we needed to set up an integration with the existing systems AS400 (ADT), Meditech (LIS), GE (RIS) to form an integrated patient electronic medical records in the clinic.

For the integration with the systems, we used the following standards of data exchange: Meditech (LIS) - HL7 v2.5.1, AS400 (ADT) - HL7 v2.3, GE (RIS) - HL7 v2.5.1 & DICOM. The problem was that these standards were highly customized within the clinic, i.e. Z-segment was actively used in HL7 messages conveying a lot of useful health care data.

Some message fields were used not for intended purposes or were redefined within the clinic scope.

Inhospital Integration Scheme

The project of integration that lasted for 5 months resulted in the following fulfilled work:

  • There were created integration modules for each system, and custom message segments were standardized, i.e. data was moved from custom fields to corresponding fields provided by the standard (where possible).
  • There were implemented international code systems: ICD-10, LOINC, SNOMED, CPT, FirstDataBank and RxNorm. For this purpose there was developed a separate module for terminology management and mapping of local vocabularies to international ones. As a result of the integration project, an integrated patient electronic medical records was formed allowing clinicians to have all necessary information about a patient and treatment for clinical decision support disposed.
